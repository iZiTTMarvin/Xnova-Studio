# 代码审查清单

审查代码时加载此参考。按顺序逐项检查 — 高优先级在前。

## 1. 正确性（严重）

- [ ] 逻辑错误：off-by-one、错误运算符、条件取反
- [ ] 空值处理：未守护的访问、缺少可选链 `?.`
- [ ] 边界情况：空输入、边界值、并发访问
- [ ] 异常处理：吞掉异常、缺少错误上下文信息
- [ ] 类型安全：隐式 `any`、未检查的类型断言
- [ ] 异步：缺少 `await`、未处理的 Promise rejection、竞态条件

## 2. 安全性（严重）

- [ ] 注入：SQL、命令、XSS — 用户输入是否已正确过滤？
- [ ] 密钥：代码中是否硬编码了 key、token、password？
- [ ] 权限：新增接口/路由是否有正确的访问控制？
- [ ] 依赖：新引入的包是否有已知漏洞？

## 3. 性能（警告）

- [ ] N+1 查询：循环中调用 DB/API → 应改为批量处理
- [ ] 不必要的重渲染：缺少 memo、引用不稳定
- [ ] 大量分配：无界数组、循环中字符串拼接
- [ ] 资源泄漏：事件监听器、定时器、订阅未释放

## 4. 可维护性（建议）

- [ ] 命名：变量/函数名是否描述了其用途？
- [ ] 复杂度：深层嵌套（>3 层）、长函数（>50 行）？
- [ ] DRY：是否有应该提取的重复逻辑？
- [ ] 注释：是否在非显而易见处解释了"为什么"而非"是什么"？

## 5. 测试

- [ ] 新逻辑是否有对应测试？
- [ ] 边界情况是否已覆盖？
- [ ] 测试是否独立（无共享可变状态）？
