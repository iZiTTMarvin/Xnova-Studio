## Affected Pages/Routes

- Desktop app launch shell — 验证默认进入空白聊天页或恢复最近项目会话，而不是落到 overview
- Project page / project session tree — 验证最近项目、最近会话、子代理树恢复逻辑
- Chat workspace view — 验证上下文条、mode 切换、subagent 状态、断连状态
- Agents page / manage agents panel — 验证 agent schema、来源、mode 过滤、编辑与删除
- Settings / Providers — 验证 TOML 读写、provider CRUD、test connection
- Settings / Memory — 验证默认开启、降级提示、关闭确认、重建入口
- Tools page — 验证 MCP / Skills 状态卡片、空态、失败态、入口跳转

## Key Interactions to Verify

- 首次打开应用时，无最近项目进入空白聊天页
- 打开已有项目时，恢复最近一次 `project / session / agent / mode / model`
- 切换 `Standard / XForge` 后，新会话默认跟随当前模式
- 自定义 agent 创建、编辑、删除后，主 Agent 选择器与 SubAgent 候选池同步更新
- `project.toml` 与 `config.toml` 合并后，项目级默认值覆盖用户级默认值
- Provider 测试连接失败时，页面显示可操作错误信息而不是静默失败
- Memory embedding 不可用时，明确显示降级为 BM25，而不是只在日志里记录
- SubAgent 在运行中、停止、部分结果三种状态下都能在聊天流和侧栏中一致呈现

## Edge Cases

- `config.toml` 缺失、损坏、部分字段缺失
- `.xnovacode/project.toml` 缺失、损坏、字段类型错误
- agent frontmatter `inherits` 指向不存在 ID
- agent `mode` 不合法或与使用场景冲突
- 最近项目路径失效、仓库分支未知、session 数据过大
- 窗口宽度缩到 `1024px` 以下时，抽屉和侧栏是否仍可用
- Bridge / runtime 尚未就绪时，用户是否看到明确状态

## Critical Paths

- 新建项目：冷启动 -> 空白聊天页 -> 绑定 workspace -> 创建首会话 -> 开始编码
- 接手已有项目：打开项目 -> 恢复最近会话 -> 恢复最近 agent/mode/model -> 继续工作
- Agent 定制：打开 Agents -> 新建 user agent -> 切换为主 Agent / SubAgent -> 发起任务
- 配置迁移：读取旧 `config.json` -> 迁移生成 `config.toml` -> 后续只写 TOML -> 旧配置不丢失
- Memory 可见性：默认开启 -> embedding 不可用 -> 明确降级提示 -> 用户仍可继续工作
